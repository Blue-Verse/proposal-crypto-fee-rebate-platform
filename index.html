핵심 요구사항 분석

  • 거래소 혜택 비교 — 거래소별 수수료, 페이백 비율, 이벤트 등 핵심 혜택 정보를 목록 및 상세 페이지 형태로 제공 (초기: Bitget, OKX)
  • 회원 관리 — 소셜 로그인(Google, Kakao) 및 이메일 가입, 마이페이지(개인정보 수정, 환급 계좌 관리)
  • 환급 신청 및 내역 관리 — 거래 내역 기반 환급액 확인, 환급 신청, 정산 내역 조회
  • 레퍼럴 링크 관리 — 거래소별 레퍼럴 코드/링크 생성 및 추적, 유입 사용자 식별
  • 관리자 기능 — 회원 관리, 거래소/혜택 정보 관리, 환급 처리/정산, 콘텐츠 관리, 통계 대시보드

제안 포인트

1. 벤치마킹 서비스 분석 기반 설계

테더맥스(tethermax.io)와 곰지네페이백(gomzipayback.com)을 철저히 분석하여, 두 서비스의 강점을 결합한 UX를 설계합니다. 테더맥스의 API 직연결 자동 정산 시스템과 곰지네페이백의 직관적인 거래소 비교 카드 UI를 참고합니다.

2. 확장 가능한 아키텍처

초기 2개 거래소(Bitget, OKX)에서 시작하여, 거래소 추가 시 설정만으로 확장 가능한 모듈형 구조를 설계합니다. 거래소 연동 어댑터 패턴을 적용하여 신규 거래소 추가 시 개발 공수를 최소화합니다.

3. 유사 프로젝트 수행 경험

Web3 dApp(Life3), P2P 금융 플랫폼, VC 펀드 관리 SaaS 등 가상화폐 및 금융 도메인에서 복수의 프로젝트를 성공적으로 완수한 경험을 보유하고 있습니다. 토큰 스왑 시스템, 자동 정산, KYC 인증, 관리자 대시보드 등 핵심 기능을 이미 구현한 바 있어 효율적인 개발이 가능합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

다양한 가상화폐 거래소의 복잡한 수수료 및 레퍼럴 혜택 정보를 사용자가 한눈에 비교하고, 본인에게 가장 유리한 조건을 선택하여 가입 및 수수료 환급을 신청할 수 있는 전문 웹 플랫폼을 구축합니다. 테더맥스, 곰지네페이백 등 기존 서비스를 벤치마킹하여, 사용자 경험과 정산 효율성에서 차별화된 MVP를 빠르게 런칭하는 것을 목표로 합니다.

2. 사용자 정의

사용자 유형설명주요 기능
일반 사용자가상화폐 거래자로서 수수료 환급을 원하는 사용자거래소 비교, 레퍼럴 가입, 환급 신청, 정산 내역 조회
관리자플랫폼 운영자회원 관리, 거래소 정보 관리, 환급 처리/정산, 통계 대시보드, 콘텐츠 관리

3. 기능 요구사항

3.1 거래소 혜택 비교

  • [필수] 거래소별 수수료율(메이커/테이커), 페이백 비율, 이벤트 정보를 카드형/테이블형으로 비교
  • [필수] 거래소 상세 페이지 (수수료 구조, 환급률, 가입 혜택, 이벤트 안내)
  • [선택] 거래소 비교 필터/정렬 (환급률순, 수수료순)
  • [선택] 환급 예상 금액 시뮬레이터 (거래량 입력 시 예상 환급액 계산)

3.2 회원 관리

  • [필수] 이메일/비밀번호 회원가입 및 로그인
  • [필수] 소셜 로그인 (Google, Kakao)
  • [필수] 마이페이지 (개인정보 수정, 비밀번호 변경)
  • [필수] 환급 계좌 등록/관리 (은행 계좌 또는 USDT 지갑 주소)

3.3 환급 신청 및 내역 관리

  • [필수] 환급 신청 폼 (거래소 선택, UID 입력, 환급 방식 선택)
  • [필수] 환급 상태 추적 (신청 → 검토 → 승인 → 지급 완료)
  • [필수] 정산 내역 조회 (기간별 필터, 거래소별 필터)
  • [선택] 자동 정산 시스템 (거래소 API 기반 환급액 자동 계산)

3.4 레퍼럴 링크 관리

  • [필수] 거래소별 레퍼럴 코드/링크 자동 생성
  • [필수] 레퍼럴 유입 사용자 추적 (가입 수, 거래량)
  • [선택] 레퍼럴 커미션 대시보드

3.5 관리자 기능

  • [필수] 회원 조회 및 관리 (검색, 필터, 상태 변경)
  • [필수] 거래소 정보 및 혜택 내용 관리 (CRUD)
  • [필수] 환급 신청 건 처리 (승인/반려) 및 정산 관리
  • [필수] 공지사항 등 콘텐츠 관리
  • [필수] 통계 대시보드 (가입자 수, 환급액, 거래소별 현황)

4. 비기능 요구사항

항목요구사항
성능API 응답 시간 300ms 이내, 페이지 로딩 2초 이내
보안HTTPS, JWT 인증, SQL Injection/XSS 방어, 환급 계좌 정보 암호화
반응형PC Web 반응형 지원 (1440px ~ 360px)
확장성거래소 추가 시 어댑터 패턴으로 최소 공수 확장
SEONext.js SSR 기반 검색엔진 최적화, 메타태그 관리

5. 기술 제약사항

  • PC Web 반응형 전용 (모바일 앱은 향후 단계)
  • 초기 연동 거래소는 Bitget, OKX 2개 — 추가 거래소는 별도 단계
  • 거래소 API 정책(호출 제한, 인증) 준수 필수
  • 환급 정산은 관리자 수동 승인 → 향후 자동화 확장 가능 설계

6. 범위 외 사항 (Out of Scope)

  • 모바일 앱 (iOS/Android) 개발
  • 3개 이상 거래소 연동 (별도 단계)
  • 실시간 자동 정산 시스템 (MVP 이후)
  • 다국어(영어) 지원 (향후 고도화)
  • 블록체인 온체인 기능 (스마트 컨트랙트 등)
